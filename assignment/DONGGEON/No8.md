# 마이크로컨트롤러(MCU)와 로봇 제어 시스템 이해

## 1. 개요
마이크로컨트롤러(MCU, Microcontroller Unit)는 로봇과 자동화 설비의 '두뇌' 역할을 수행하며, 환경 인식, 데이터 처리, 액추에이터 제어를 담당합니다. 본 문서는 로봇 제어의 핵심 원리와 인터페이스, 통신 방식, 실시간 제어 및 아두이노 활용에 대해 정리합니다.

## 2. 로봇 제어 방식
로봇 제어의 핵심은 **피드백 루프(Feedback Loop)**입니다. MCU는 다음 과정을 초당 수천 번 반복하며 로봇을 제어합니다.

- **센싱(Input):** 센서(초음파, 엔코더 등)를 통해 현재 상태를 측정합니다.
- **연산(Processing):** PID 알고리즘 등을 사용하여 목표값과 현재값의 차이를 계산합니다.
- **출력(Output):** 모터 드라이버나 서보 모터에 제어 신호를 전송합니다.

## 3. 기본 제어 방식 및 인터페이스
MCU와 주변 장치 간의 소통을 위한 주요 I/O 인터페이스입니다.

| 구분 | 신호 성격 | 주요 용도 |
| :--- | :--- | :--- |
| **Digital I/O** | 0 또는 1 (High/Low) | 스위치 상태 확인, 릴레이 On/Off |
| **Analog I/O** | 전압 세기 (ADC 변환) | 센서 데이터(온도, 압력 등) 수집 |
| **PWM** | 펄스 폭 변조 | 모터 속도 제어, LED 밝기 제어 |

## 4. 통신 방식 (데이터 전송)
시스템 구성 요소 간의 데이터 교환을 위한 표준 프로토콜입니다.

- **UART:** 비동기 직렬 통신. PC 디버깅 및 블루투스 모듈 연결에 주로 사용.
- **I2C:** 동기식 2선 통신. 다수의 센서를 단일 버스에 연결할 때 유용.
- **SPI:** 고속 동기식 통신. 디스플레이 및 대용량 데이터 전송에 최적화.

## 5. 실시간 제어와 타이머
로봇의 정밀한 제어를 위해 시간의 결정론적(Deterministic) 확보가 중요합니다.

- **타이머의 역할:** CPU 클럭을 기준으로 정확한 시간 간격으로 인터럽트(Interrupt)를 발생시켜 제어 루프(예: 1ms PID 제어)를 유지합니다.
- **사용 절차:** 클럭 설정 → 카운터 값 지정 → 인터럽트 활성화 → ISR(인터럽트 서비스 루틴) 구현 → 시작.

## 6. 아두이노(Arduino)의 개념과 활용
아두이노는 하드웨어와 소프트웨어가 결합된 프로토타이핑 생태계입니다.

- **개념:** MCU 보드 + 개발 환경(IDE) + 오픈소스 라이브러리.
- **현장 활용 전략:** 현장 설비의 신뢰성은 PLC가 담당하고, 아두이노는 보조적인 센서 모니터링, 데이터 로거(Data Logger), 또는 아이디어 검증(PoC) 단계에서 활용하는 것이 효과적입니다.
